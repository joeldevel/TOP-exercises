A (function) object constructor creates an object when used with new keyword

function Person(name, age){ this.name = name; this.age = age; this.makeBread = function() { return `${this.name} makes bread!`; } } let ana = new Person('ana', 28); let floncho = new Person('floncho', 56); console.log(ana.name); // -> ana console.log(ana.age); // -> 28 console.log(ana.makeBread()); // -> ana makes bread! console.log(floncho.makeBread()); // -> floncho makes bread!

The prototype

[[Prototype]] references either null or another object and allows one object to inherit properties from another

If a property is no found in an object, JS will go up in the prototype chain looking for that property

__proto__ is a getter/setter for the [[Prototype]] property

Every time a new object is created, all functions inside are copied for each instance

To avoid that we can attach these functions to the pototype

function Person(name, age){ this.name = name; this.age = age; } //put the function in the prototype Person.prototype.makeBread = function() { return `${this.name} makes bread!`; }
